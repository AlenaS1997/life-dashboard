### 3. n8n переезд — главный bloodbath

Состоялась самая муторная часть всей сессии. Хронология:

**3.1 OAuth callback URL = localhost.** При создании Google credential в
n8n обнаружили что Redirect URL `http://localhost:5678/...`, а не
публичный домен Railway. Google не пустит OAuth с таким callback.

Решение: добавить env vars в Railway → Variables:
- `N8N_HOST=n8n-production-e175.up.railway.app`
- `N8N_PROTOCOL=https`
- `WEBHOOK_URL=https://n8n-production-e175.up.railway.app/`
- `N8N_EDITOR_BASE_URL=https://n8n-production-e175.up.railway.app/`

**3.2 База n8n слетает при рестарте.** После применения env vars Railway
сделал redeploy → SQLite-база n8n (хранится в `/home/node/.n8n`) слетела
вместе с эфемерным диском. Workflow и credentials исчезли. Алёна попала
на owner setup ещё раз.

Решение: добавить **Persistent Volume** через Canvas Railway (правый
клик на сервисе → Attach Volume → mount path `/home/node/.n8n`).

**3.3 Volume permission denied.** После добавления volume контейнер
крашился с `EACCES: permission denied, open '/home/node/.n8n/config'`.
Volume mount даёт root-owned папку, а n8n работает под user `node`.

Решение: для скорости откатили volume (приняли риск что данные снова
слетят при будущем рестарте — на практике это редко). Долгосрочно
надо либо использовать Postgres backend, либо custom start command с
`chown`.

**3.4 OAuth client в Google Cloud — новый wizard.** Пришлось пройти
многошаговый «Project configuration» (Branding / Audience /
Contact / Finish). Тип External, добавить email в Test users. Потом
создать OAuth client ID типа Web application с Authorized Redirect URI
= наш Railway URL.

**3.5 Client Secret скрыт.** В новом интерфейсе Google после Create
client секрет не показан в UI. Нужно либо «Add Secret», либо
«Download JSON» — секрет будет в JSON-файле.

**3.6 Google Sheets — отдельный credential.** В n8n Calendar и Sheets
требуют **разные** credential типы (`Google Calendar OAuth2 API` vs
`Google Sheets OAuth2 API`). Один и тот же OAuth client используется,
но в n8n это два отдельных credentials. Создали оба под основным
аккаунтом savchenko.

**3.7 Календарь Fluffy — другой Google аккаунт.** События нужно тянуть
из календаря руководителя Алёны (`Fluffyismylifemoscow@gmail.com`),
у Алёны нет пароля от него. Решение: календарь уже расшарен с её
основным аккаунтом → используем существующий credential, просто
указываем `calendar_id = Fluffyismylifemoscow@gmail.com` в Конфиг ноде.

**3.8 Параметры Calendar ноды не мигрировали.** При импорте JSON в n8n
параметры `timeMin` / `timeMax` / `singleEvents` / `orderBy` не
подхватились в новые поля `After` / `Before` / `Recurring Event Handling`
/ `Order By`. Пришлось вручную:
- After = `{{ $now.startOf('day') }}`.
- Before = `{{ $now.endOf('day') }}`.
- Recurring Event Handling = **All Occurrences**.
- Timezone = `Europe/Moscow`.

После этого нода вернула 19 events за сегодня (в Fluffy-календаре).

**3.9 IF-нода не пропускала items от Маппим к Append.** В моём дизайне
workflow данные шли: Маппим (19) → Очистить (1) → IF → Append (1 from
IF). Append получал result от Очистить, не 19 events от Маппим.

Решение для скорости: убрали IF и Очистить, прямой pipeline
Маппим → Append. Дубли при ежедневных запусках разрулим в следующую
сессию через `appendOrUpdate` operation.

**3.10 Workflow опубликован и активирован.** В новой версии n8n кнопка
называется «Publish» (вместо тумблера Active/Inactive). Алёна нажала
Publish — workflow стал активным с расписанием cron.

- В `.env.example` добавлены `FATSECRET_CONSUMER_KEY` и `_SECRET`.
