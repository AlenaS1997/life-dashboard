## Что я сделал ночью

### 1. Полировка дашборда

- **Цветовая индикация good/warn/bad** на карточках «Сегодня». Рамка
  карточки и точка в углу светятся зелёным, жёлтым или красным в
  зависимости от значения. Пороги: сон ≥7 ч хорошо, <6 плохо; HRV ≥40
  хорошо, <30 плохо; шаги ≥8000 хорошо; стресс ≤40 хорошо, ≥60 плохо;
  Body Battery min ≥40 хорошо.
- **Сравнение с прошлой неделей** в блоке «Тренды». На каждой метрике
  бейджик `▲ 8 %` или `▼ 5 %`, цвет зелёный = улучшение, красный =
  ухудшение (для стресса инверсия — меньше = лучше). Если разница <3 % —
  серый бейджик `≈ 1 %`.
- **Heatmap сна за 30 дней** — отдельная секция, сетка 7 колонок,
  цвет фиолетового от тёмного (мало сна) к яркому (много). Hover/тап
  показывает дату и часы.
- **Pull-to-refresh** на iPhone. Потяни страницу вниз с самого верха —
  появится подсказка «Потяни ещё чуть-чуть… → Отпусти — обновится».
  Триггерит `load()` без перезагрузки страницы.
- **Service Worker bumpнут до v3** — старый кэш чистится при первой
  загрузке новой версии, ничего вручную делать не надо.

### 2. Telegram-бот: подготовка к активации (промпт v3 — «аналитик»)

По твоему запросу — больше анализа, меньше клише, рекомендации только на
основе подмеченных связей.

- **Системный промпт v3** переписан под аналитика:
  - Структура: *Вчера · Тенденция (7/14/30) · Что заметил · Рекомендация
    · Погода · Сегодня · Дашборд.*
  - **Тенденция** — конкретные дельты 7/14/30 дней + streak'и (сколько
    подряд дней метрика хуже базы).
  - **Что заметил** — связи между метриками и календарём (вечерние
    встречи → сон → HRV).
  - **Рекомендация** — ровно ОДНА строка, конкретная, на основе данных.
    Если связи не нашлось — блок пропускается («лучше тишина, чем
    шаблонный совет»).
  - Жёсткий запрет на «учебные» советы («больше воды»).
  - История версий v1 → v2 → v3 — в `n8n/claude-prompt.md`.
- **Code-нода переписана**. Теперь она считает:
  - `trends` — avg за 7/14/30 дней + дельты 7d vs prev_week и 7d vs 30d.
  - `streaks` — сколько подряд дней метрика хуже 30-дневной базы.
  - `daily_last_7` — массив за 7 дней с метриками + счётчиками
    `events_evening` (после 18:00) и `events_late` (после 21:00).
- **Новая нода «Календарь за 7 дней + сегодня»** (Google Calendar getAll)
  тянет события за -7..+1 дней, чтобы Claude мог делать выводы вида
  «4 дня подряд встречи после 21:00 → сон <7 ч и HRV ниже».
- **Погода** в дайджесте + ссылка на дашборд — как в v2.
- **max_tokens** поднят с 700 до 1500 для развёрнутого анализа.

⚠️ Важное последствие: для morning-digest теперь нужен Google OAuth
credential (тот же, что для gcal-sync). Если не подключишь — нода
календаря упадёт, но `continueOnFail: true` на ней — workflow продолжит
работу без анализа причин (только тренды и рекомендации без календарной
привязки). Так что технически без Google можно жить, но v3 во всей красе
работает только когда Google credential есть.

## Шаг 4 — активация Telegram-бота (30–40 минут)

Это финальный большой кусок проекта. Делай **один раз** — потом каждое
утро будешь получать дайджест от своего бота.

### 4.1 Создаём Telegram-бота через BotFather (3 мин)

1. В Telegram открой `@BotFather` (это официальный бот, проверь синюю
   галочку у имени).
2. Команда: `/newbot`
3. На вопрос «Alright, a new bot. How are we going to call it?» — введи
   имя (видно в чате), например: **`Life Dashboard`**
4. На вопрос про username — введи уникальный `@<что-то>_bot`, например:
   **`@alena_life_bot`**. Если занято — добавь цифры.
5. BotFather пришлёт сообщение с **HTTP API token** вида
   `1234567890:ABCdef...` — скопируй и сохрани, понадобится в шаге 4.4.
6. **ВАЖНО:** в Telegram открой только что созданного бота → жми кнопку
   **Start** внизу. Без этого бот не сможет писать тебе в личку.

### 4.2 Узнаём свой chat_id (1 мин)

1. В Telegram открой `@userinfobot`.
2. Нажми **Start** — он сразу пришлёт сообщение с
   `Id: 123456789`. Это число и есть твой `chat_id`. Сохрани.

### 4.3 Получаем Anthropic API key (5 мин)

1. Открой https://console.anthropic.com/
2. Залогинься (или зарегистрируйся, если нет аккаунта).
3. Слева в меню **API Keys** → **Create Key** → имя `Life Dashboard n8n`.
4. Скопируй ключ вида `sk-ant-api03-...` — он показывается **один раз**.
5. Если у тебя trial-кредиты не настроены — в **Plans & Billing** добавь
   способ оплаты. Утренние дайджесты — копейки, $5 хватит на месяцы.

### 4.4 Подключаем credentials в n8n (5 мин)

Открой https://n8n-production-e175.up.railway.app

**Telegram credential:**
1. Слева **Credentials** → `+` → поиск «Telegram» → **Telegram API**.
2. Поле `Access Token` — вставь токен от BotFather (`1234567890:ABCdef...`).
3. Имя credential: **Life Dashboard Bot**.
4. **Save**.

**Anthropic credential:**
1. **Credentials** → `+` → поиск «Header Auth» → **Header Auth**.
2. Поля:
   - `Name` (имя credential): **Anthropic x-api-key**
   - `Header Name`: **`x-api-key`**
   - `Header Value`: твой `sk-ant-api03-...`
3. **Save**.

### 4.5 Импортируем morning-digest workflow (5 мин)

1. **Workflows** → `+` → **Import from File** → выбери
   `n8n/morning-digest-workflow.json` (Desktop → LifeDashboard → n8n).
2. Откроется схема из 8 нод. На нодах **«Claude: сочиняет дайджест»**
   и **«Отправить в Telegram»** будут красные ⚠️ — credentials
   ещё не привязаны.
3. Кликни в **«Claude: сочиняет дайджест»** → справа поле
   `Credential to connect with` → выбери **Anthropic x-api-key** → закрой.
4. Кликни в **«Отправить в Telegram»** → поле `Credential` → выбери
   **Life Dashboard Bot** → закрой.
5. Кликни в ноду **«Конфиг»** → найди поле `telegram_chat_id` → замени
   `REPLACE_ME_WITH_YOUR_CHAT_ID` на свой `chat_id` из 4.2.
6. **Save** (вверху справа или Cmd+S).

### 4.6 Тест (2 мин)

1. В правом верхнем углу — кнопка **Execute workflow**. Жми.
2. Все 8 нод должны позеленеть.
3. Проверь Telegram — должен прийти дайджест от твоего бота.

**Если не пришёл:**
- Открой раздел **Executions** слева → последний run → посмотри какая
  нода упала. Самые частые проблемы:
  - **`Тянем события дня` пустой** → `gcal_events` ещё не записан Make-ом.
    Не страшно, дайджест уйдёт без событий.
  - **`Claude` 401** → неправильный API key.
  - **`Claude` 529 / 429** → временная перегрузка Anthropic. Retry уже
    встроен, подожди и повтори.
  - **`Telegram` 400 chat_not_found** → ты ещё не нажала Start у бота
    (вернись в 4.1 пункт 6).
- Пришли мне скрин ошибки — разберёмся.

### 4.7 Активация (1 мин)

Когда тест прошёл успешно — в правом верхнем тумблер **Active** → ON.
Теперь каждое утро в **08:30 МСК** (после Garmin sync в 08:05 и
Make/n8n gcal sync в 08:00) дайджест будет приходить в Telegram сам.
